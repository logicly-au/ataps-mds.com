Statistical Linkages Key (SLK)

    Overview

    This item has been mandatory for new referrals as of 1st June, 2013.  If you have any difficulties implementing this change please email us at support@ataps-mds.com.

    A new field has been introduced in the Patients record of the MDS in September 2012. This new field is labelled ‘SLK’, an acronym for ‘Statistical Linkage Key’. The purpose of the SLK is to maintain greater accuracy of patient data, within and across Primary Health Networks. The SLK provides the capacity to identify individual clients across time in order to support more accurate reporting of repeat vs new clients being serviced. This capacity also helps managers to allocate resources according to program requirements (e.g. limits to the maximum number of sessions per client per 12 month period).

    It is important to note that an SLK is not a unique key for each patient since it is possible for a derived SLK to exactly match the derived SLK of another patient. This will only occur in a very small percentage of patients, which allows the SLK to be used for reporting purposes but not when an exact match is required. i.e. searching for a specific patient within the ATAPS MDS.

    The SLK feature requires organisations to enter the following information into an SLK Generator within the ATAPS MDS: Family Name, First Name, Date of Birth, Gender. It is important to note that the name information will NOT be stored, it will simply serve as the combination of data from which an SLK will be generated for this patient. This SLK produced will be in de-identified form (i.e., a code). The conventions of this process are documented on the Australian Institute of Health and Welfare website:

    http://meteor.aihw.gov.au/content/index.phtml/itemId/349510

    Privacy Concerns

    Concerns were raised that the SLK includes enough of a person's details (parts of their name, their gender, and date of birth) that they may still be identifiable. In order to alleviate this the SLK can now be provided, and is always stored as a meaningless looking string of characters via a 'hashing' algorithm similar to those used by secure computer systems to protect user passwords. For example the SLK:

    RONOH010119701

    gets hashed to:

    25YA2351JSRF1CVT9CWRQXTHRBXFTTNB

    This hashed SLK can then be compared in the same way the non-hashed SLK can.

    The SLK can now be entered or uploaded in its hashed form. Instructions for generating SLKs for upload can be found on the Upload File Format page.

    Any data entered into the SLK generator is not stored online, it remains within the privacy of the end user's computer.

    Instructions on using the SLK in the MDS

    1. From within the Patients record, click on the ‘SLK’ link. This will open up the SLK generator.
    2. Enter the following details: Family Name, First Name, Date of Birth, Gender.
    3. Click on the ‘Generate SLK’ button.
    4. An SLK should appear in the SLK field in your patient’s record.

    Furthermore, a standalone SLK generator is now available at https://ataps-mds.net/slk.  This does not require users to log in to the MDS to use this.

    Instructions for uploading the SLK

    Instructions for generating SLKs for upload can be found on the Upload File Format page.

    Test data to allow developers to test their implementation of the SLK specification can be found on the SLK Test Data Set page.
