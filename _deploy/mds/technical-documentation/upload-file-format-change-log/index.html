Upload File Format Change Log

    Changes

    Changes on 26/4/2015

    An update was made on the 26/4/2014 to improve  uploads of multiple sessions on the same day.  

    • A new field "org_ses_id" has been added to the session upload format.  This field is optional. Primary Health Networks may supply an identifier (up to 64 characters) to uniquely identify the session.

    Changes on 29/7/2013

    The following change was made on the 29/7/2013:

    • All headers in uploaded files are converted to lower case and trimmed of whitespace before being further processed.
    • All column data is whitespace trimmed before any other processing.

    Changes on 20/6/2013

    An update was made on the 20/6/2013 to further align the MDS with the Operational Guidelines. Acute Mental Health Team has been added as a referrer for the Suicide Prevention Program.

    Changes on 17/5/2013

    Updates were made on the 17/5/2013 to align the MDS with the Operational Guidelines. In the tables below please refer to changelog entries dated 17/5/2013 for details of the changes. A summary is given below:

    • Child program - Diagnostic categories, referral sources and session-types have been aligned with the ATAPS Operational Guidelines.
    • ATSI Suicide Prevention has been added as a program type.
    • Extreme Climatic Events (ECE): program types have been added for Tier 2 ECE - Bushfire, Tier 2 ECE - Flood/Cyclone and Tier 2 ECE - Other.
    • Existing ECE program types have been retired and existing referrals mapped to the new program types.
    • Options for all other programs have been aligned with the ATAPS Operational Guidelines.

    Where possible changes have been made backwards compatible with previous upload formats:

    • Referrals uploaded with old ECE program types are mapped to the new ECE program types.
    • Sessions uploaded with old Session Types are mapped to the new Session Types.

    Retired fields (such as st_family_therapy) may be supplied for referrals and sessions dated on or before 17/5/2013. An error is thrown if the referral/session date is after this and that field has a value.

    Changes from V3 in September 2009

    Please be aware that the version 3 format files are still accepted by the upload system. 

    • Columns can be given in any order.
    • 'extra_start' column is not required when supplying optional data fields.
    • The various repeated 'id' columns are not neccessary:
      • patients.csv: id and division.
      • referrals.csv: division, id, gp, referral
      • sessions.csv: division, id, referral, ahpro
    • The 'gp_name' column is now named 'referrer_code'. (Old name still accepted)
    • The 'referrer' column is now named 'referrer_type'. (Old name still accepted)
    • session_modality is now named 'modality' for consistency. (Old name still accepted)
    • eng_level is now named 'english_level' for consistency. (Old name still accepted)
    • Some columns which used a numerical reference to a foreign key can now use the name as well. (eg. You can put '1' or 'General ATAPS' in the referral_type field. For the referrer_type field you could put '7' or 'Midwife') This is consistent with the rest of the foreign-key fields, which all used the name rather than number.
    • DASS21 and DASS42 outcome measures have dropped the total fields - So you need to delete the columns dass21_1, dass21_2, dass42_1, dass42_2
