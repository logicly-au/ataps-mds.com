Upload File Format

This document details the file format required to upload data to the MDS.

If you are a third-party software developer and would like access to test your uploads please follow the instructions in the FAQ.

    All files format info

    File format: CSV (Comma separated values)

    Character encoding: ASCII encoding.

    Line endings: CR or CRLF or LF.

    When uploading data, the MDS requires all core column headings to be present, even if some of these may not contain data. The separately documented extra fields may be included as required. The order of the columns is not important. Column headings are case insensitive. 

    Patients upload format

    Uploaded patient data is cumulative, that is, new patients are added to the patients stored in the database. By default the upload will fail if it includes a patient who has previously been uploaded. By selecting the "Replace duplicates" tickbox you can force the upload to replace existing patient details with the newly uploaded details.

    Referrals upload format

    Referrals are handled the same as Patients, the data is cumulative and duplicates are not allowed unless the "Replace duplicates" option is checked. This will remove duplicate data that is in the MDS, and replace it with the data being uploaded.

    NOTE: MHPro and Referrer codes must be unique. You cannot assign the same code for an MHPro and a Referrer.
