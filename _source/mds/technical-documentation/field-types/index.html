Field Types

    Base Field Types

    Boolean

    • yes
    • no
    • 1
    • 0
    • true
    • false

    Date

    • YYYY-MM-DD (eg. 1980-12-31)
    • YYYY/MM/DD (eg. 1980/12/31)
    • DD-MM-YYYY (eg. 31-12-1980)
    • DD/MM/YYYY (eg. 31/12/1980)

    The regular expression for validating dates is m/\d{4}[-\/]\d{2}[-\/]\d{2}/a

    The date cannot be before the 1st of January, 1900.

    Identifier

    Up to 64 plain ASCII characters.

    Integer

    Should be fine for all positive and negative integer values in the ranges likely to be used by this application. Contact us before using values exceeding +/- 2 billion.

    Number

    Up to 6 digits before the decimal point; up to 2 digits after the decimal point.

    Past Date

    Same as the Date field type but the date must be on or before tomorrow.

    Postcode

    Four digit number in the range 0000-9999.

    SLK

    • 14 character SLK as defined at http://meteor.aihw.gov.au/content/index.phtml/itemId/349510,
    • a Crockford encoded sha1 hash of a 14 character SLK. This must be 32 characters in length. The Crockford encoding spec is defined at http://www.crockford.com/wrmg/base32.html.
    • a hex encoded sha1 hash of a 14 character SLK. This must be 40 characters in length.

    SLK values are stored in sha1_hex format. SLK values uploaded in other formats will be converted to this format before being stored. All download files will contain a sha1_hex encoded SLK.

    It is important to note that an SLK is not a unique key for each patient since it is possible for a derived SLK to exactly match the derived SLK of another patient. This will only occur in a very small percentage of patients, which allows the SLK to be used for reporting purposes but not when an exact match is required. i.e. searching for a specific patient within the ATAPS MDS.

    Text

    Plain ASCII printable characters.

    FK - Referrer or MHPro code

    Plain ASCII printable characters.

    Minimum Length:
    2
    Maximum Length:
    50

    Codes must be unique. Therefore, once a code is used within the MDS, the same code cannot be allocated again, to a different individual. i.e If a code has been used for a referrer it cannot then be used for an MHPro.
